## THE TERMINAL OBJECTIVE

> **Epistemic precision as infrastructure.**
>
> Every routing decision in the AION brain architecture serves one terminal objective: making AI systems and human-AI collaboration more epistemically precise — more accurate about what is known, what is reasoned, what is strategic, and what is unverified.

This is not a mission statement written for external audiences. It is the operational intent document. It is what Auftragstaktik (F30) calls the commander's intent — the principle that fills routing gaps when specific instructions are unavailable or outdated.

---

## WHAT EPISTEMIC PRECISION MEANS IN PRACTICE

`[D]` Epistemic precision means knowing the certainty level of every claim that exits the system. Not "this is probably right" — but "this is `[D]` directly measured" or "this is `[R]` logically derived from measured evidence" or "this is `[?]` unverified and must be treated as such."

`[R]` A system that routes outputs without tracking their epistemic provenance is a system that can confabulate — generate plausible-sounding outputs that have no verified basis, presented with the same confidence as outputs that do. Routing architecture that serves epistemic precision is routing architecture that makes confabulation structurally difficult.

`[?]` Whether any single routing decision fully serves epistemic precision is always an open question. The mission principle is a direction, not a destination. The architecture moves toward it permanently.

## HOW THIS DOCUMENT IS USED

**Case 1 — Ambiguous routing input:**
An input arrives that does not classify cleanly into the INPUT-TAXONOMY. Route to the destination that best serves epistemic precision for this input type. If uncertain — route to AMYGDALA for assessment rather than routing incorrectly.

**Case 2 — Competing routing options:**
Two valid routing options exist. Both satisfy Principles 1–5. Apply Principle 6: which option better serves epistemic precision? That option is correct.

**Case 3 — Proposed routing rule without clear justification:**
A new routing rule is proposed. It cannot trace clearly to one of the six core principles. Apply the mission principle test: does this rule make the system more epistemically precise, or less? If less — do not add it. If uncertain — do not add it until certainty is established.

**Case 4 — Routing rule that once served the mission but no longer does:**
Conditions have changed. A routing rule that was correct under previous conditions produces worse epistemic outcomes under current conditions. The mission principle is the authority to remove it — not the six core principles, which govern the form of routing rules, but the mission principle, which governs their purpose.
