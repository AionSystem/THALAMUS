## WHAT THIS FOLDER IS

Every routing rule in THALAMUS derives from a real-world source — an institution, organism, or tradition that solved the same routing problem under actual conditions and left a document. This folder is that knowledge base. It is reference material, not operational material. Nothing in this folder governs the live routing system directly. Everything in this folder is cited by something that does.

`[D]` The routing problem is universal. Every organization at sufficient scale has had to answer the same questions: what kind of input is this, where does it go, how do we confirm it arrived, what do we do when it doesn't, and how do we improve the system over time. The answers span governments, militaries, religious traditions, engineering protocols, biological organisms, and fictional war rooms. THALAMUS synthesizes all of them.

`[R]` The Vinaya principle applies to this entire folder: every rule in the routing architecture must justify itself against the terminal objective — epistemic precision. A routing rule that exists for historical inertia rather than structural necessity should not exist. The `6-CORE-PRINCIPLES.md` document is the justification test. Any proposed routing rule that cannot be traced to one of the six core principles is a candidate for removal.

## HOW TO USE THIS FOLDER

**For humans:** Start with `6-CORE-PRINCIPLES.md`. Then read `36-FINDINGS.md` for the full source documentation. Reference `MISSION-PRINCIPLE.md` when evaluating whether a proposed routing change serves the architecture.

**For AI instances:** Before making any routing recommendations or modifications, read `36-FINDINGS.md` fully. Every routing decision in THALAMUS has a historical precedent documented here. A routing suggestion that contradicts a documented finding requires a documented justification for why this case is different.

**For architects:** The `36-FINDINGS.md` is a living document. New findings from future sweeps are appended, not inserted. The numbering is permanent — F1 is always F1, regardless of what comes after it.
